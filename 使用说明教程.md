# 猫棒助手使用说明教程

本工具用于通过 SSH 配置 GPON 猫棒，常见用途是写入原光猫认证信息、设置 VLAN、查看注册状态、测试拨号、获取日志和升级固件。

## 快速说明

这部分对应软件状态页右侧的“使用说明”，适合第一次配置时按顺序操作。

1. 先进入原光猫后台，记录 GPON SN、LOID、LOID 密码或 PLOAM Password，以及上网 VLAN、IPTV VLAN 等认证数据。
2. 打开软件后，顶部连接区一般保持默认 `192.168.1.10`、端口 `22`、用户 `root`，填写猫棒 SSH 密码后点击“连接”，再点击“读取”。
3. 进入“快速设置”标签页，填写从原光猫获取到的数据。
4. 点击底部“保存并应用”，等待约 8 秒后点击“重启”，然后等待猫棒重启完成，建议等待 3 分钟。
5. 重新连接软件，点击“刷新状态”，查看 PON 状态。显示 `O5` 代表猫棒已经注册成功。
6. 注册成功后，在路由器里设置 PPPoE 拨号上网。

## 使用前准备

需要准备以下信息：

- 猫棒管理地址：通常是 `192.168.1.10`。
- SSH 登录信息：通常用户为 `root`，密码以当前固件为准。
- 原光猫认证信息：GPON SN、LOID、LOID CheckCode 或 PLOAM Password。
- VLAN 信息：上网 VLAN 填 PVID，IPTV 如不用可以留空。
- 宽带拨号账号和密码：用于路由器拨号，或用于软件里的拨号测试。

注意：

- GPON SN 通常在原光猫背面标签或原光猫后台里。
- 联通、电信常见为 LOID 认证；移动常见为 PLOAM Password 认证。
- 不确定的高级参数不要随意改，尤其是 MIB、OMCID、OMCC、固件升级相关功能。

## 顶部连接区

顶部连接区用于连接猫棒和切换页面模式。

- Host：猫棒 IP，默认 `192.168.1.10`。
- Port：SSH 端口，默认 `22`。
- User：SSH 用户，默认 `root`。
- Password：SSH 密码。
- 连接：连接猫棒。
- 读取：读取猫棒当前配置，并同步刷新状态。
- 重启：向猫棒发送重启命令。
- 熟练模式：关闭时显示新手常用页面；开启后显示更完整的高级配置页面。

底部“保存并应用”用于提交本次修改。软件只会保存你改动过的项目，保存后部分设置需要重启才会完全生效。

## 状态页

状态页用于判断猫棒是否正常注册、光功率是否正常、当前固件和设备型号是什么。

主要信息：

- PON 状态：最关键的注册状态。`O5` 表示认证成功；非 `O5` 通常需要检查 SN、LOID、PLOAM、VLAN 或线路状态。
- Signal：光纤信号检测。
- RX / TX Power：接收和发射光功率。
- Die / Laser Temp：芯片和激光器温度。
- ETH Speed：网口速率。
- Active Bank：当前启动分区。
- IOP Version：当前系统版本。
- Model：设备型号，在线升级会用它匹配固件。
- OMCID Version、GPON SN、LOID、PLOAM Password：当前读取到的认证信息。

状态页默认暂停实时刷新。需要实时观察时，点击状态页顶部“实时状态（已暂停，点击恢复）”，软件会每 3 秒刷新一次；再点击一次可暂停。

## 快速设置

快速设置是新手最常用页面，只保留认证和关键 VLAN。

填写建议：

- GPON SN：必须填写，通常填原光猫的 GPON SN 或 SN。
- LOID：联通、电信用户常用。
- LOID CheckCode：如果运营商有 LOID 密码，就填这里；没有可留空。
- PLOAM Password：移动用户常用，一般为 10 位 ASCII 字符。
- PVID：上网 VLAN，只填原光猫 Internet 连接名称或后台里对应的 VLAN 数字。
- IPTV 下行组播 VLAN：不用 IPTV 可以留空。

LOID 和 PLOAM Password 不要混填。软件会尽量互斥处理，避免同时写入两种认证方式。

## 基础认证

基础认证页是熟练模式下的认证信息完整页面，项目和快速设置基本一致。

适合以下情况：

- 想单独确认或修改 GPON SN。
- 想单独修改 LOID、LOID CheckCode。
- 想单独修改 PLOAM Password。
- 快速设置无法满足，需要更清楚地分项检查认证数据。

修改后点击底部“保存并应用”，再根据需要重启猫棒。

## VLAN 相关设置

VLAN 页用于处理上网 VLAN、IPTV VLAN、VLAN 转换和兼容模式。

常用项目：

- 互操作兼容模式：大部分 VLAN 设置依赖它，修改后通常需要重启。
- PVID：上网 VLAN。范围一般为 `1-4094`；如果运营商是 untag 模式，可填 `u`。
- 强制创建 PVID 规则：PVID 写了但不生效时可尝试。
- 强制创建 ME 规则：PON 已经 `O5`，但无法拨号时可尝试。
- 下行组播 VLAN：填写原光猫中的组播 VLAN。
- 下行组播 VLAN 转换：把组播 VLAN 转换为 IPTV 认证 VLAN。
- VLAN 转换/绑定：适合复杂 VLAN 转换场景。
- N:1 VLAN 转换模式：多个用户侧 VLAN 转为同一个网络侧 VLAN 时使用。
- VLAN 脚本调试日志：排查 VLAN 问题时使用。

保存前软件会检查 VLAN 范围和明显冲突。如果 PVID、下行组播 VLAN、下行组播 VLAN 转换三者填成相同值，软件会阻止保存。

## 组播设置

组播页主要用于 IPTV 组播异常排查。

- 组播版本设置：可选择 IGMP 版本，基于 IPoE 的 IPTV 组播异常时可尝试调整。
- 强制创建 ME309 规则：IPTV 组播异常时可尝试启用。

如果不用 IPTV，这一页通常不用改。

## 拨号测试

拨号测试用于在 Windows 电脑上直接测试 PPPoE 拨号，判断猫棒注册和 VLAN 是否已经满足上网条件。

使用方法：

1. 确保猫棒 PON 状态已经是 `O5`。
2. 填写宽带账号和宽带密码。
3. 点击“拨号”。
4. 查看拨号日志和结果。
5. 测试结束后点击“断开”。

说明：

- 这个功能调用 Windows 本机拨号能力，只适合 Windows 环境。
- 如果猫棒已经 `O5` 但拨号失败，优先检查 PVID、强制 ME 规则、宽带账号密码和运营商绑定信息。
- 最终日常使用一般还是在路由器里拨号。

## 高级自定义设置

高级自定义页风险较高，只建议明确知道用途时使用。

功能包括：

- 自定义 MIB 配置文件：可填写厂商 ID、设备 ID、ONT 版本。
- 自定义 OMCID 版本：用于模拟特定原光猫软件版本。
- 恢复原版 OMCID：撤销自定义 OMCID。
- 自定义 OMCC 版本：默认一般为 `160`。
- 禁用光纤状态检测、启用发光状态检测、禁用 RX_LOS 报告、忽略 RX_LOSS 消息。
- 启用 TTL 控制台、OMCID 日志重定向到 TTL、OMCID 日志级别设置。

风险提醒：

- MIB、OMCID、OMCC 改错可能导致认证失败、反复重启，严重时可能需要救砖。
- OMCID 版本长度过长会被软件阻止，因为超长字符串可能损坏设备文件。
- 不确定时不要开启自定义 OMCID。

## 自动维护

自动维护页用于设备异常时自动恢复。

常用项目：

- 重启 OpenWrt：接入光纤后，如果长时间非 `O5`，尝试自动重启系统。
- 重启前等待次数、重启次数：控制等待和重试次数，一般建议 `5-10`。
- 重启 LCT 接口：管理接口不可达时尝试恢复。
- 追踪 IP：用于 Ping 检测，建议填局域网内稳定 IP。
- 保存调试日志：重启前保存日志到设备。
- 直接重启 OpenWrt：达到指定等待时间后直接重启。

这类功能适合无人值守环境。普通一次性配置时通常不用开启。

## 高级功能

高级功能页在新手模式下可见，主要用于读取更多信息和执行维护操作。

功能说明：

- 刷新：读取 RX / TX 光功率、OLT 类型、VLAN 信息、上次重启原因。
- 一键日志获取：从猫棒采集调试日志并保存到电脑，适合排查问题或发给售后。
- 一键重置：恢复出厂并重启，IP 会设置为 `192.168.1.10`。此操作会清除配置，谨慎使用。
- 双 IP 访问后台：新增第二个管理 IP，方便在不改原管理地址的情况下访问后台。
- 启用 VLAN：给第二管理 IP 绑定 VLAN。不要和上网 VLAN、IPTV VLAN 冲突。
- 克隆原光猫 MAC：把原光猫 HOST MAC 写入猫棒，适合运营商绑定 MAC 的场景。
- 做电信版 G7615 下行光设备：特殊用途功能。使用时必须注意光功率，接收光太强可能烧毁猫棒。

双 IP 注意：

- 老固件中 `192.168.1.x` 常被管理口占用，第二 IP 建议使用其它网段。
- 某些旧系统必须启用 VLAN 才能启用第二 IP。

## 在线升级

在线升级会根据状态页读取到的设备型号匹配固件。

使用步骤：

1. 先连接猫棒，并在状态页点击“刷新状态”，确保 Model 已经读出。
2. 进入“在线升级”页，点击“在线固件检测”。
3. 在下拉框选择固件版本。
4. 点击“刷写并重启”。
5. 刷写过程中不要断电，不要拔猫棒，不要断开光猫供电。
6. 等待设备重启完成后重新连接检查状态。

软件会尽量校验固件 MD5。如果 MD5 不匹配或没有记录，软件会弹窗提醒；不确定时不要继续刷写。

## 本地固件升级

本地固件升级用于刷写电脑上的 `.image` 固件文件。

使用步骤：

1. 点击“浏览”，选择正确型号的 `.image` 固件。
2. 点击“计算 MD5”，和固件发布方提供的 MD5 比对。
3. 确认固件型号、版本无误后点击“刷写并重启”。
4. 刷写过程中不要断电。
5. 等待设备重启完成后重新连接检查状态。

刷错固件可能导致设备变砖。本地升级前必须确认固件适配自己的猫棒型号。

## 常见判断

- PON 状态为 `O5`：认证成功，可以继续做拨号测试或路由器拨号。
- PON 状态非 `O5`：先检查 GPON SN、LOID、PLOAM Password、VLAN 和光纤信号。
- 有光功率但无法 `O5`：多半是认证信息、OMCI 兼容或运营商绑定问题。
- 已经 `O5` 但无法拨号：重点检查上网 VLAN、强制 ME 规则、宽带账号密码。
- 软件连接不上：检查电脑 IP 是否在同一网段、猫棒 IP 是否为 `192.168.1.10`、SSH 是否开启、密码是否正确。

## 安全提醒

- 保存配置前确认数据来自原光猫，不要凭猜测填写。
- 刷机、恢复出厂、高级自定义前先确认设备型号和用途。
- 刷写固件时不要断电。
- 光功率过强可能损坏猫棒，做下行光相关功能时尤其要注意。
- 不需要 IPTV 时，IPTV 相关 VLAN 可以留空，减少误配置。
